# C++ 学习路径选择指南

## 两种学习路径

本项目提供两种学习路径，你可以根据当前水平选择：

| 路径 | 适合人群 | 时间 | 目标 |
|------|----------|------|------|
| **路径 A：快速强化** | 已有 C++ 基础的高级开发者 | 12 周 | 专家级（Staff/Principal） |
| **路径 B：系统进阶** | 仅具备基础编程能力 | 6-12 个月 | 计算机专家级别 |

---

## 路径 A：快速强化（12 周）

**适用人群**：
- 已有 2+ 年 C++ 开发经验
- 熟悉面向对象、STL、基本并发
- 目标是成为架构师/技术专家

**学习路径**：详见 `DAILY_PLAN.md`

**核心特点**：
- 能力诊断 → 补洞 → 项目驱动
- 12 个 Skills 覆盖全能力域
- 每周递进产出
- 领域级项目收尾

---

## 路径 B：系统进阶（6-12 个月）

**适用人群**：
- 仅具备基础编程能力
- 了解简单变量、循环、条件判断
- 无系统 C++ 知识体系
- 未接触过面向对象、工程化

**核心逻辑**：基础夯实 + 领域拆解 + AI 赋能 + 阶梯式实践

### 阶段一：基础夯实与 AI 工具适配（3-4 周）

**核心目标**：
- 快速补齐 C++ 基础语法
- 掌握 AI 工具基础使用
- 建立"AI+基础学习"模式

**核心内容**：
1. 基础语法（数据类型、变量、运算符、输入输出）
2. 流程控制（if-else、switch、for、while）
3. 数组与指针（一维/二维数组、指针基础）
4. 函数基础（定义、声明、调用、参数传递、重载）
5. 面向对象入门（类与对象、封装、成员变量/函数）

**AI 工具选型**：

| 工具类型 | 推荐工具 | 应用场景 |
|----------|----------|----------|
| 知识点讲解 | 豆包、ChatGPT | 通俗语言讲解、答疑 |
| 代码生成 | CodeGeeX4、豆包代码助手 | 生成基础代码、排查 bug |
| 练习批改 | 豆包、C++ 在线判题 AI | 生成练习题、批改作业 |

**AI 使用技巧（基础版）**：
```
"我是 C++ 基础学习者，用最简单的语言讲解指针的用法，
结合 1 个简单代码示例，不要太复杂"
```

### 阶段二：核心领域进阶（18-22 周）

6 大核心领域分层突破，每个领域遵循"基础→核心→实践→项目"流程。

#### 领域 1：C++ 基础进阶（3-4 周）

**核心内容**：
- 指针进阶（指针数组、函数指针、空指针/野指针）
- 函数进阶（模板、递归、内联）
- 面向对象深化（构造/析构、拷贝构造、继承、多态）
- STL 入门（vector、list、sort、find）

**AI 辅助方法**：
```
"用最简单的语言讲解函数指针的用法，结合 2 个基础代码示例，
注释详细，避免复杂逻辑"
```

**阶段目标**：
- 掌握指针进阶、函数进阶
- 能独立编写面向对象基础代码
- 能使用 STL 常用容器
- 能排查基础代码 bug

#### 领域 2：C++ 高级编程特性（3-4 周）

**核心内容**：
- C++11-14：智能指针、lambda、range-based for、constexpr
- C++17：结构化绑定、optional、filesystem
- C++20：Concepts、Ranges、协程基础
- STL 进阶：容器底层实现、算法逻辑、使用陷阱

**AI 辅助方法**：
```
"讲解 shared_ptr 的基础用法，生成 3 个简单代码示例，
注释详细，说明如何避免内存泄漏"
```

**阶段目标**：
- 熟练使用 C++11-20 核心特性
- 能结合场景选择合适特性
- 能规避基础使用陷阱

#### 领域 3：C++ 底层原理（4-5 周）

**核心内容**：
- 内存管理（堆/栈/静态区、new/delete 原理、内存泄漏）
- 面向对象底层（虚函数表、多态原理、继承内存布局、this 指针）
- 高级特性底层（智能指针引用计数、移动语义）
- 编译器基础（预处理、编译、汇编、链接、符号表）

**AI 辅助方法**：
```
"用最简单的语言讲解虚函数表的实现机制，结合 1 个基础代码示例，
用文字描述虚函数表的结构，避免复杂术语"
```

**阶段目标**：
- 理解核心底层原理
- 能通过底层原理分析代码问题
- 能手写简单智能指针模拟实现

#### 领域 4：设计模式与架构设计（4-5 周）

**核心内容**：
- 设计模式（单例、工厂、观察者、适配器）
- 架构原则（SOLID）
- C++ 架构入门（分层架构、事件驱动、接口设计）
- 设计优化（高内聚低耦合）

**AI 辅助方法**：
```
"设计一个简单的日志输出功能，推荐合适的设计模式，
生成基础代码，说明设计思路"
```

**阶段目标**：
- 熟练运用常用设计模式
- 能根据场景设计合理架构
- 能评审并优化简单架构

#### 领域 5：工程化开发（2-3 周）

**核心内容**：
- 工程化工具（CMake、GDB、Valgrind）
- 代码规范（命名、注释、审查流程）
- 项目构建与部署（编译、打包、Git）
- 调试与优化（bug 排查、性能优化）

**AI 辅助方法**：
```
"生成一个简单 C++ 项目的 CMakeLists.txt，
详细说明每一步配置的含义，指导我完成配置"
```

**阶段目标**：
- 独立完成工程搭建、配置、调试、部署
- 熟练使用基础工程化工具
- 遵循基础编码规范

#### 领域 6：跨领域应用（2-3 周）

**核心内容**：
- 跨领域应用（TCP/UDP 服务器、嵌入式基础）
- 操作系统基础（进程/线程、内存管理、锁机制）
- 数据结构与算法（红黑树、哈希表、动态规划、贪心）
- 语言对比（C++ vs C vs Java）

**AI 辅助方法**：
```
"生成一个简单的 TCP 服务器代码，注释详细，
说明核心模块的作用，适合基础学习者"
```

**阶段目标**：
- 独立完成简单场景项目
- 理解 C++ 与底层知识关联
- 具备基础跨领域应用能力

### 阶段三：综合实践与复盘（6-8 周）

#### 阶梯式项目实践（4-6 周）

**项目 1：学生管理系统（1-2 周）**
- 覆盖领域：1、2、4
- 功能：增删改查、文件存储
- 技术：面向对象、单例模式、工程化搭建

**项目 2：简单高性能服务器（3-4 周）**
- 覆盖领域：2、3、4、5
- 功能：高并发、日志输出、连接管理
- 技术：工厂/观察者模式、工程化部署、性能优化

**AI 辅助项目实践**：
- 需求分析：梳理需求，生成需求文档
- 架构设计：评审架构，指出耦合问题
- 问题解决：排查复杂问题
- 优化提升：全面优化代码

#### 复盘与总结（2 周）

- 知识复盘：生成知识图谱
- 项目复盘：总结优缺点、经验教训
- 短板补漏：针对性补漏计划
- 思维沉淀：形成基础方法论

### 阶段四：专家级突破（长期）

#### 专家级能力突破（8-10 周）

- 底层原理深化（ABI 兼容、编译器优化、STL 源码分析）
- 设计模式与架构进阶（23 种模式、分布式/微服务架构）
- 复杂项目实践（分布式服务器、嵌入式控制系统）
- 行业复杂问题挑战（高并发优化、内存泄漏排查、跨平台兼容）

#### 持续进阶（长期）

- 技术迭代跟进（C++23/26 特性、行业前沿）
- 知识输出与复盘（技术文章、总结）
- 技术交流与提升（模拟面试、技术评审）

## AI 使用核心禁忌

### ⚠️ 关键提醒

1. **不依赖 AI 直接生成代码交差**
   - 基础阶段：重点用 AI 辅助理解知识点、生成示例
   - 进阶阶段：用 AI 排查问题、优化代码
   - 核心逻辑和设计必须自己掌握

2. **AI 回答可能存在错误**
   - 底层原理、复杂架构尤其需要验证
   - 进阶阶段必须自己验证（运行代码、查阅权威文档）
   - 不盲目相信 AI

3. **重点用 AI 解决效率和难度问题**
   - 效率问题：梳理知识、排查基础 bug
   - 难度问题：拆解复杂原理
   - 不是"替代思考"
   - 专家级核心是"独立思考、解决复杂问题"

4. **基础阶段避免直接学习复杂内容**
   - 依托 AI 循序渐进
   - 不急于求成
   - 确保每个知识点扎实掌握

## 整体时间规划

| 阶段 | 时间 | 核心目标 | AI 核心作用 |
|------|------|----------|------------|
| 基础夯实 | 3-4 周 | 补齐 C++ 基础 | 讲解、示例、答疑 |
| 核心领域 | 18-22 周 | 6 大领域突破 | 拆解难点、案例、排查 |
| 综合实践 | 6-8 周 | 融会贯通 | 辅助开发、复盘优化 |
| 专家突破 | 长期 | 达成专家级 | 拆解原理、解决复杂问题 |

## 路径选择建议

### 选择路径 A（12 周）如果：
- 已有 2+ 年 C++ 经验
- 熟悉 STL、面向对象、基础并发
- 目标是成为架构师/技术专家
- 能每天投入 3-4 小时

### 选择路径 B（6-12 个月）如果：
- 仅具备基础编程能力
- 未系统学习过 C++
- 需要从基础语法开始
- 能每天投入 2-3 小时
