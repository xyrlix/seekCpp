## 示例卡片

### 知识卡片 #001

## 基本信息
- **创建日期**: 2026-01-15
- **知识域**: 内存性能
- **难度**: ★★★☆☆
- **来源**: 代码审查

## 知识点
**标题**: 移动语义的陷阱：移动后对象状态

## 问题描述
使用 `std::move` 后，原对象处于有效但未定义状态，访问其值会导致 bug。

## 错误代码
```cpp
std::string s1 = "hello";
std::string s2 = std::move(s1);
std::cout << s1.size(); // 危险！移动后 s1 的状态未定义
```

## 根本原因
`std::move` 只是将对象转为右值引用，实际移动发生在移动构造函数中。
移动后，源对象应处于有效但未定义状态，通常为空。

## 正确做法
```cpp
std::string s1 = "hello";
std::string s2 = std::move(s1);
// 不再使用 s1，或重新赋值
s1 = "new value"; // 安全
std::cout << s1.size(); // OK
```

## 关键要点
1. 移动后不要访问原对象值
2. 移动构造函数应保证源对象可安全析构
3. 需要继续使用对象时，应拷贝而非移动

- [ ] 静态分析工具检查 use-after-move
- [ ] 单元测试覆盖移动语义

## 知识卡片分类

### 按错误类型
- **内存错误**: 泄漏、悬垂、越界、use-after-free
- **并发错误**: 竞态条件、死锁、原子性破坏
- **逻辑错误**: 边界条件、类型转换、运算符优先级
- **设计错误**: 紧耦合、过度设计、接口不清晰
- **性能问题**: 不必要拷贝、缓存不友好、算法复杂度高

### 按知识域
- **语言基础**: 值类别、对象模型、模板推导
- **标准库**: 容器、算法、迭代器失效
- **内存管理**: RAII、智能指针、移动语义
- **并发编程**: 线程、锁、原子、内存序
- **架构设计**: 接口、组件、设计模式

## 间隔复习算法

### 复习周期
```
第 1 天: 学习新卡片
第 3 天: 第一次复习
第 7 天: 第二次复习
第 14 天: 第三次复习
第 30 天: 第四次复习
第 60 天: 第五次复习（长期记忆）
```

### 复习方法
1. **自测**: 不看答案，尝试回忆
2. **编码**: 重新实现相关代码
3. **讲解**: 向 AI 讲解这个知识点
4. **应用**: 在实际代码中应用

### 掌握度评估
- **★ 未掌握**: 完全忘记，需要重新学习
- **★★ 模糊**: 记得概念，但细节不清
- **★★★ 基本**: 能理解，但应用不熟练
- **★★★★ 熟练**: 能正确应用
- **★★★★★ 精通**: 能讲解给别人，能发现相关 bug
