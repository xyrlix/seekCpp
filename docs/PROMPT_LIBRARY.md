# AI 提示词库

## 使用说明

本提示词库包含 5 类高效提示词，对应 AI 的 5 个角色。直接复制使用即可。

---

## 一、诊断类提示词（诊断师角色）

### 1.1 能力诊断
```
请扮演 C++ 能力诊断师，通过 20 个问题定位我在以下 8 个能力域的盲区：
1. 语言基础（值类别、对象模型、模板）
2. 标准库（容器、算法、现代特性）
3. 内存性能（RAII、缓存、profiling）
4. 并发编程（线程、原子、无锁）
5. 系统工程（构建、链接、CI）
6. 架构设计（接口、组件、模式）
7. 网络编程（Socket、协议、框架）
8. 规范判断（Core Guidelines、审查）

要求：
- 每域 2-3 题
- 简要回答即可
- 根据回答生成能力雷达图
- 输出个性化补洞计划
```

### 1.2 知识缺口分析
```
请基于以下我的回答，分析知识缺口：

[粘贴你的回答]

要求：
1. 按优先级排序缺口
2. 每个缺口说明：
   - 为什么重要
   - 不掌握的后果
   - 推荐学习资源
3. 生成 12 周补洞计划
```

### 1.3 代码诊断
```
请提供 3 段有 bug 的 C++ 代码，覆盖：
1. 内存管理问题
2. 并发安全问题
3. 异常安全问题

让我找出问题，然后：
- 评分（0-10）
- 指出遗漏
- 解释原理
```

## 二、讲解类提示词（讲解员角色）

### 2.1 四版本讲解
```
请用四种方式解释 [概念名称]：

1. 直觉版（50 字）
   - 用大白话讲本质
   - 类比生活中的例子

2. 正式版（200 字）
   - 严格定义和规则
   - 标准中的描述

3. 源码版（300 字）
   - 底层实现原理
   - 关键源码分析

4. 反例版（200 字）
   - 常见错误用法
   - 错误后果
   - 正确做法
```

### 2.2 概念拆解
```
请把 [主题] 拆成我必须掌握的 20 个概念：

要求：
1. 按依赖关系排序（先学什么后学什么）
2. 每个概念包含：
   - 一句话定义
   - 为什么重要
   - 10 分钟自测题
3. 标注难度（1-5 星）
```

### 2.3 苏格拉底式教学
```
请用苏格拉底式方法教我 [主题]：

规则：
1. 每次只问一个问题
2. 等我回答后再问下一个
3. 根据我的回答调整难度
4. 引导我自己推导出结论
5. 不要直接给答案

从最基础的概念开始。
```

## 三、审查类提示词（审稿人角色）

### 3.1 五维度审查
```
请扮演 C++ 核心库审查员，针对以下代码从五个维度指出问题：

```cpp
[粘贴代码]
```

审查维度：
1. 对象生命周期（悬垂引用、临时对象）
2. 异常安全（资源泄漏、强保证）
3. 线程安全（竞态、死锁、原子性）
4. 性能（不必要的拷贝、缓存不友好）
5. 可读性（命名、复杂度、意图）

输出格式：
- [CRITICAL] 必须修复（附位置、问题、修复方案）
- [WARNING] 建议修复（附位置、问题、修复方案）
- [INFO] 可选优化（附位置、问题、优化方案）
```

### 3.2 Core Guidelines 检查
```
请基于 C++ Core Guidelines 检查以下代码：

```cpp
[粘贴代码]
```

要求：
1. 列出所有违反的规则编号
2. 说明为什么违反该规则
3. 给出修复方案
4. 按严重程度排序
```

### 3.3 现代 C++ 重写
```
请只用标准库重写这个模块：

```cpp
[粘贴代码]
```

要求：
1. 使用 C++17/20 现代特性
2. 说明为什么这样更现代
3. 说明为什么这样更安全
4. 说明为什么这样更易测试
```

### 3.4 设计审查
```
请审查这个系统设计：

[描述设计]

审查点：
1. 接口边界是否清晰
2. 依赖关系是否合理
3. 错误处理策略是否一致
4. 是否可测试、可替换、可扩展
5. 指出耦合点和改进建议
```

## 四、出题类提示词（出题人角色）

### 4.1 递进练习生成
```
请针对 [主题] 生成递进练习：

要求：
1. 7 天训练计划
2. 每天 3 个练习，从易到难：
   - 基础题（概念理解）
   - 实现题（代码编写）
   - 架构题（设计决策）
3. 每个练习包含：
   - 题目描述
   - 输入输出示例
   - 验收标准
   - 提示（可选）
```

### 4.2 Bug 场景生成
```
请给我 5 个真实的 bug 场景：

要求：
1. 覆盖不同错误类型：
   - 内存泄漏
   - 竞态条件
   - 异常不安全
   - 逻辑错误
   - 性能问题
2. 每个场景包含：
   - bug 代码片段
   - 触发条件
   - 期望行为
3. 让我写单元测试证明修复
```

### 4.3 面试题生成
```
请生成 20 道 C++ 面试题：

要求：
1. 难度分布：
   - 简单 5 题（基础概念）
   - 中等 10 题（理解深度）
   - 困难 5 题（系统设计）
2. 覆盖领域：
   - 语言特性
   - 标准库
   - 并发编程
   - 内存管理
   - 架构设计
3. 每道题附：
   - 考点说明
   - 期望答案要点
   - 追问问题
```

## 五、复盘类提示词（复盘器角色）

### 5.1 错误整理
```
以下是我本周遇到的 10 个错误：

[粘贴错误列表]

请：
1. 分类错误类型（内存/并发/逻辑/设计）
2. 每个错误生成知识卡片：
   - 错误描述
   - 根本原因
   - 正确做法
   - 预防策略
3. 按频率排序
```

### 5.2 间隔复习计划
```
请基于以下知识卡片生成间隔复习计划：

[粘贴知识卡片]

要求：
1. 使用间隔重复算法：
   - 第 1 天复习
   - 第 3 天复习
   - 第 7 天复习
   - 第 14 天复习
   - 第 30 天复习
2. 每次复习出 3 道测试题
3. 标记薄弱卡片重点复习
```

### 5.3 周总结生成
```
请基于以下我的本周活动生成周总结：

[粘贴本周完成的任务、代码、错误]

总结包含：
1. 本周完成清单
2. 掌握的知识点
3. 待改进项
4. 下周重点
5. 能力成长对比
6. 推荐下周学习资源
```

### 5.4 知识图谱生成
```
请基于我过去 [N] 周的学习生成知识图谱：

[粘贴学习记录]

要求：
1. 可视化的知识点关系图
2. 标注已掌握/待加强/未学习
3. 指出知识盲区
4. 推荐学习路径
```

## 高效使用技巧

### 1. 分段输出
当内容较多时，要求 AI 分段输出：
```
"请分 3 次输出，每次输出一部分内容，我回复'继续'后再输出下一部分"
```

### 2. 限定格式
要求结构化输出：
```
"请用表格/列表/JSON 格式输出"
```

### 3. 迭代优化
不满意时要求重写：
```
"这个解释不够清晰，请用更简单的语言重新解释，并增加一个例子"
```

### 4. 对比分析
要求对比不同方案：
```
"请对比方案 A 和方案 B 的：
- 性能差异
- 安全性差异
- 可维护性差异
- 适用场景"
```

### 5. 源码分析
要求深入底层：
```
"请分析 [函数/类] 的 GCC/Clang 实现源码：
- 关键数据结构
- 核心算法
- 优化策略"
```
