## 训练目标

通过 12 周高强度刻意练习，从**基础编程能力**进阶到**C++ 专家级**，具备：

1. **语言深度理解** - 不仅会用，更理解设计动机和代价
2. **工程实践能力** - 能独立设计、实现、优化中型系统
3. **架构判断力** - 能在性能、抽象、可读性间做权衡
4. **代码审查能力** - 能发现别人方案里真正会出问题的地方
5. **持续学习能力** - 在新领域快速建立知识图谱并落地

## Skills 使用指南

### 已创建的 12 个 Skills

| Skill 名称 | 用途 | 触发场景 |
|------------|------|----------|
| `cpp-diagnostic-assessor` | 能力诊断评估 | 开始训练前评估水平 |
| `cpp-basics-trainer` | 基础语法训练 | 变量、指针、引用练习 |
| `cpp-stl-master` | STL 专项训练 | 容器、算法、迭代器 |
| `cpp-memory-raii` | 内存与 RAII | 智能指针、资源管理 |
| `cpp-template-meta` | 模板与元编程 | 泛型编程、SFINAE |
| `cpp-concurrency` | 并发编程 | 线程、锁、原子 |
| `cpp-performance` | 性能优化 | 分析、基准、缓存 |
| `cpp-modern-features` | 现代特性 | C++17/20/23 新特性 |
| `cpp-system-engineering` | 系统工程 | 构建、链接、CI |
| `cpp-architecture` | 架构设计 | 接口、组件、模式 |
| `cpp-network-programming` | 网络编程 | Socket、协议、框架 |
| `cpp-code-reviewer` | 代码审查 | 质量检查、反馈 |

### 如何使用 Skills

#### 方法 1：自动触发
直接描述任务，AI 会自动匹配对应 Skill：

```
你："我想练习智能指针"
AI：自动调用 cpp-memory-raii Skill
```

#### 方法 2：手动指定
明确要求使用某个 Skill：

```
你："使用 cpp-concurrency 训练我"
AI：加载并发编程训练内容
```

#### 方法 3：组合使用
复杂任务可组合多个 Skill：

```
你："用 cpp-architecture 设计一个线程池，再用 cpp-performance 优化"
AI：先加载架构设计，再加载性能优化
```

## 训练流程

### 完整训练周期

```
开始
  ↓
[诊断评估] ← cpp-diagnostic-assessor
  ↓ 生成能力地图
[制定计划] ← 个性化 12 周计划
  ↓
[每日训练] ← 学→练→查→复盘
  ↓
[每周产出] ← 项目/组件/报告
  ↓
[阶段评估] ← 对比初始能力地图
  ↓
[调整计划] ← 针对薄弱点强化
  ↓
[最终项目] ← 领域级系统
  ↓
专家级 ✓
```

### 每日训练节奏

| 时间 | 活动 | 时长 | 使用 Skill |
|------|------|------|------------|
| 早上 | 概念学习 | 45min | 对应领域 Skill |
| 中午 | AI 问答 | 30min | cpp-code-reviewer |
| 晚上 | 代码实现 | 90min | 对应领域 Skill |
| 周末 | 项目/复盘 | 2-3h | 全部相关 Skills |

### 每周四步循环

1. **学习** - 看小主题，建立模型
2. **产出** - 写代码，必须有真实产出
3. **校验** - AI 审查，打分反馈
4. **复盘** - 整理知识卡，间隔复习

## 12 周计划

### Week 1-2: 诊断补洞
- **重点**: 值类别、对象生命周期、异常安全、模板基础
- **产出**: 个人能力缺口地图
- **每日任务**: 详见 `training/daily/week01-02/`

### Week 3-4: 标准库现代化
- **重点**: ranges、span、optional、variant、expected
- **产出**: 2 个小组件重写版
- **每日任务**: 详见 `training/daily/week03-04/`

### Week 5-6: 内存与性能
- **重点**: RAII、对象布局、benchmark、profiling
- **产出**: 1 次性能优化报告
- **每日任务**: 详见 `training/daily/week05-06/`

### Week 7-8: 并发编程
- **重点**: jthread、atomic、任务队列、同步
- **产出**: 1 个并发服务原型
- **每日任务**: 详见 `training/daily/week07-08/`

### Week 9-10: 架构设计
- **重点**: 模块化、接口边界、错误模型、测试
- **产出**: 1 个中型项目骨架
- **每日任务**: 详见 `training/daily/week09-10/`

### Week 11-12: 领域项目
- **重点**: 选真实方向做完整交付
- **产出**: 可展示的专家级项目
- **每日任务**: 详见 `training/daily/week11-12/`

## AI 工作流

### 5 个 AI 角色

| 角色 | 功能 | 示例提示词 |
|------|------|------------|
| **诊断师** | 定位知识盲区 | "问我 20 个问题找出我的 C++ 盲区" |
| **讲解员** | 四版本讲解 | "用直觉/正式/源码/反例讲这个概念" |
| **审稿人** | 代码审查打分 | "从 5 个维度审查我的代码" |
| **出题人** | 递进练习生成 | "给我从易到难的 7 天训练" |
| **复盘器** | 知识卡生成 | "把这周错误整理成复习卡片" |

### 高效提示词模板

#### 诊断类
```
"请扮演 C++ 诊断师，问我 20 个问题定位我在对象模型、
模板、并发、性能上的盲区，每题要求简要回答即可。"
```

#### 讲解类
```
"请用四种方式解释 [概念]：
1. 直觉版：用大白话讲本质
2. 正式版：严格定义和规则
3. 源码版：底层实现原理
4. 反例版：错误用法和后果"
```

#### 审查类
```
"请扮演 C++ 核心库审查员，针对这段代码从以下维度指出问题：
1. 对象生命周期
2. 异常安全
3. 线程安全
4. 性能
5. 可读性
并给出最小修改方案。"
```

#### 出题类
```
"请把这个主题拆成我必须掌握的 20 个概念，
按依赖关系排序，并给每个概念设计一个 10 分钟自测题。"
```

#### 复盘类
```
"以下是我本周遇到的 10 个错误，请：
1. 分类错误类型
2. 生成知识卡片
3. 制定间隔复习计划
4. 出 5 道复习测试题"
```

## 评估标准

### 专家级指标

| 指标 | 达标标准 |
|------|----------|
| 设计动机理解 | 能解释特性的为什么这样设计 |
| 代码安全 | 不查资料写出安全可维护实现 |
| 权衡能力 | 在性能/抽象/可读性做合理权衡 |
| 审查能力 | 指出别人方案里真正会出问题的地方 |
| 知识图谱 | 在新领域快速建立知识图谱并落地 |
