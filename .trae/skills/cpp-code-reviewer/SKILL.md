# C++ 代码审查系统

## 概述

本 Skill 提供专业的 C++ 代码审查服务，从多个维度分析代码质量，提供具体的改进建议。模拟高级工程师的代码审查流程。

## 审查维度

### 1. 内存安全（最高优先级）

#### 检查项：
- 内存泄漏（new 无 delete、循环引用）
- 悬空指针/引用
- 数组越界
- 未初始化变量
- use-after-free
- double-free

#### 审查提示词：
```
"请重点检查内存安全问题：
1. 是否有内存泄漏
2. 指针生命周期是否正确
3. 是否存在悬空引用
4. 数组访问是否越界"
```

### 2. 资源管理

#### 检查项：
- 是否使用 RAII 管理资源
- 文件句柄是否正确关闭
- 锁是否正确释放
- 智能指针选择是否合理
- 自定义删除器是否需要

#### 审查提示词：
```
"请检查资源管理：
1. 文件/网络/锁是否正确释放
2. 是否应该使用智能指针
3. RAII 设计是否合理"
```

### 3. 异常安全

#### 检查项：
- 异常发生时资源是否安全
- 异常安全级别是否满足需求
- try-catch 是否合理
- noexcept 是否正确标注

#### 审查提示词：
```
"请分析异常安全性：
1. 异常发生时是否有资源泄漏
2. 强异常安全保证是否满足
3. 异常处理是否合理"
```

### 4. 设计质量

#### 检查项：
- 类设计是否合理（单一职责）
- 接口是否清晰
- 是否过度设计
- 依赖关系是否合理
- 是否遵循 SOLID 原则

#### 审查提示词：
```
"请审查代码设计：
1. 类职责是否单一
2. 接口设计是否合理
3. 是否有过度设计
4. 依赖注入是否需要"
```

### 5. 性能优化

#### 检查项：
- 不必要的拷贝
- 容器选择是否最优
- 算法复杂度是否合理
- 缓存友好性
- 内存分配优化

#### 审查提示词：
```
"请分析性能问题：
1. 是否有不必要的拷贝
2. 容器/算法选择是否最优
3. 内存分配是否可以优化"
```

### 6. 现代 C++ 实践

#### 检查项：
- 是否使用 C++11/14/17/20 特性
- auto 使用是否合理
- range-based for 是否使用
- lambda 表达式是否恰当
- std::optional/variant 是否需要

#### 审查提示词：
```
"请检查现代 C++ 实践：
1. 是否有更现代的写法
2. 是否可以使用 auto
3. 是否需要 std::optional/variant"
```

## 审查流程

### Step 1: 静态分析
- 语法检查
- 编译警告分析
- 代码规范检查

### Step 2: 动态分析
- 内存泄漏检测
- 性能分析
- 边界条件测试

### Step 3: 设计审查
- 架构合理性
- 接口设计
- 可维护性

### Step 4: 改进建议
- 优先级排序
- 具体重构方案
- 代码示例

## 反馈格式

### 严重问题（必须修复）
```
[CRITICAL] 内存泄漏风险
位置：server.cpp:45
问题：new 分配的内存在异常路径下未释放
建议：使用 std::unique_ptr 管理该资源
```

### 中等问题（建议修复）
```
[WARNING] 不必要的拷贝
位置：client.cpp:78
问题：函数参数按值传递导致拷贝
建议：改为 const reference 或 std::move
```

### 轻微问题（可选优化）
```
[INFO] 代码风格
位置：main.cpp:12
问题：可以使用 auto 简化类型声明
建议：auto it = container.begin();
```

## 代码示例

### 有问题代码
```cpp
void processData() {
    int* data = new int[100];
    // ... 处理数据
    if (error) return;  // 内存泄漏！
    delete[] data;
}
```

### 修复后代码
```cpp
void processData() {
    auto data = std::make_unique<int[]>(100);
    // ... 处理数据
    if (error) return;  // 自动释放
    // 不需要手动 delete
}
```
