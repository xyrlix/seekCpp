# C++ 专家级进阶训练

## 概述

本 Skill 面向已有高级 C++ 基础的开发者，通过**领域驱动学习**（Domain-Oriented Learning）快速跨越到专家级（Staff/Principal Level）。

**核心理念**：将 AI 作为"资深架构师搭档"和"底层原理透视镜"。

## 训练阶段

### 阶段一：深挖底层原理与标准演进（理论专家）

#### 1.1 C++20/23/26 特性透视

**训练方法**：
- 不要直接看文档
- 给 AI 一段 C++11/14 旧代码
- 让 AI 重构为 C++20/23 风格（Ranges, Coroutines, Concepts）
- 分析重构后对内存、二进制体积、运行效率的影响

**AI 练习任务**：
```
"解释 C++20 Coroutines 的无栈协程在底层是如何通过堆分配
（或优化掉堆分配）实现的，并对比 Go 的有栈协程。"
```

**工具配合**：
- 使用 Compiler Explorer (https://godbolt.org/)
- 将代码丢给 AI 解释汇编差异
- 分析编译器优化效果

#### 1.2 内存模型与并发对抗

**AI 深度对练 - 教中学模式**：
```
"请扮演一个对 std::atomic_thread_fence 或 
memory_order_release/acquire 有误解的初级开发者，
我通过反驳和代码演示来纠正你。"
```

**训练目标**：
- 多线程可见性问题
- 指令重排原理
- 缓存一致性协议
- False Sharing 问题

### 阶段二：系统架构与设计模式（设计专家）

#### 2.1 架构拆解

**训练方法**：
- 选择知名开源项目（LLVM, Chromium, RocksDB）
- 让 AI 梳理核心设计模式应用

**练习**：
```
"RocksDB 为什么选择 LSM-Tree 而不是 B+ Tree？
请从 C++ 内存管理和磁盘 I/O 的角度分析。
如果我要用 C++23 重构它的写路径，你会推荐哪些新特性来降低延迟？"
```

#### 2.2 AI 驱动的权衡分析

**AI 对话模板**：
```
"我现在要设计一个支持每秒 100 万次请求的低延迟量化交易撮合引擎，
请对比基于 Actor 模型和共享内存无锁队列两种设计在 C++ 中的
实现难度和性能预期。"
```

**分析维度**：
- 延迟预期
- 吞吐量
- 实现复杂度
- 可维护性
- 扩展性

### 阶段三：领域深度应用（开发与应用专家）

#### 3.1 影子项目

**项目方向**：
- 分布式键值存储
- 高性能协程调度器
- 简单深度学习推理引擎
- 低延迟交易系统

**AI 角色 - 严格 Code Review**：
```
"这是我实现的无锁队列代码，请从以下维度进行严格审查：
1. 缓存行伪共享（False Sharing）
2. 指令重排
3. 异常安全性
并给出优化建议。"
```

#### 3.2 Vibe Coding 进阶

**专家级标志**：
- 能通过自然语言精准定义复杂的系统约束
- 能一眼识别 AI 生成代码中的隐蔽陷阱
  - 隐式拷贝
  - 生存期陷阱
  - 内存泄漏
  - 竞态条件

**训练方法**：
- 用自然语言描述系统需求
- 让 AI 生成代码
- 审查并指出问题
- 手动修复和重构

## 高级训练原则

### 1. 拒绝浅层回答

当 AI 给出代码时，追问：
- "为什么不选择替代方案？"
- "这个实现会导致二进制膨胀吗？"
- "编译器会优化这段代码吗？"
- "在极端并发下会有什么问题？"

### 2. 验证 AI 的局限性

- 故意给 AI 喂入有 Memory Leak 的代码
- 故意给 AI 喂入有 Race Condition 的代码
- 测试 AI 是否能发现
- 培养对代码安全性的极端敏感度

### 3. 输出倒逼输入

- 利用 AI 梳理知识点
- 在技术社区发表深度技术专栏
- Stack Overflow 回答
- 公司内部分享

## 高级提示词模板

### 底层分析
```
"请分析以下代码在 x86-64 架构下的汇编输出：
[代码]
重点关注：
1. 编译器是否进行了内联优化
2. 是否有不必要的内存访问
3. 循环是否被向量化
```

### 架构对比
```
"请对比以下两种架构在以下场景的优劣：
场景：[具体场景描述]
方案 A：[描述]
方案 B：[描述]
对比维度：
1. 延迟
2. 吞吐量
3. 内存占用
4. 实现复杂度
5. 可维护性
```

### 性能诊断
```
"以下是 Perf 分析报告：
[粘贴报告]
请推测：
1. CPU 瓶颈可能在哪里
2. 是否有 Cache Miss 问题
3. 是否有 Branch Misprediction
4. 优化建议
```

### 跨语言对比
```
"请对比 C++ 和 Rust 在以下方面的底层实现差异：
[具体特性]
从以下角度分析：
1. 内存模型
2. 所有权机制
3. 零成本抽象
4. 运行时开销
```
