# 代码安全审计规范
## 审计范围
- 语言相关漏洞：C/C++（缓冲区溢出、use-after-free、内存泄漏）；Python（注入、反序列化、路径遍历）
- 通用漏洞：SQL注入、XSS、CSRF、权限绕过、敏感信息泄露
- 网络安全：协议安全、加密实现、认证授权、输入校验
- 第三方依赖：漏洞扫描、版本更新、许可证合规

## 审计流程
1. 信息收集：架构、依赖、接口、数据流向
2. 静态代码分析：SonarQube、Semgrep、Clang Static Analyzer、Bandit
3. 手动审计：逻辑漏洞、边界条件、权限校验、加密实现
4. 动态测试：单元测试、模糊测试、渗透测试
5. 漏洞分级与修复：高危优先，提供修复方案与验证方法
6. 复测与合规：修复后验证，输出审计报告

## 安全编码要求
- 输入校验：所有外部输入严格校验（类型、范围、格式）
- 最小权限：进程、账户、API 权限最小化
- 加密：敏感数据传输/存储加密（TLS、AES、RSA）
- 日志：不记录敏感信息，日志可审计、不可篡改
- 依赖管理：定期扫描漏洞，及时更新
