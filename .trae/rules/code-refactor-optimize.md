# 代码重构与优化规范
## 重构原则
- 不改变外部行为：重构前后功能完全一致
- 小步快走：每次重构小范围，及时测试
- 测试先行：重构前保证测试用例完备
- 工具辅助：静态分析、重构工具、自动化测试

## 常见重构模式
- 重命名：变量/函数/类名更清晰
- 提取函数/类：拆分长函数、大模块
- 合并重复代码：消除重复，统一入口
- 简化条件：扁平化if-else、替换为多态/查表
- 优化数据结构：提升访问/修改性能
- 解耦：减少依赖，提高模块化

## 性能优化流程
1. 性能基准：建立benchmark，量化当前性能
2. 瓶颈定位：profiling（perf、gprof、cProfile、Py-Spy）
3. 优化：针对性优化（内存/CPU/IO）
4. 验证：对比基准，确保提升且功能正确
5. 文档：记录优化点、原因、效果

## 工程要求
- 重构后必须通过所有测试
- 代码审查：重点关注逻辑正确性、可读性
- 持续优化：定期review，避免代码腐化
