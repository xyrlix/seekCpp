# 生产级代码全局基线
## 核心原则
- 可读性 > 简洁性 > 性能（性能瓶颈再优化）
- 所有代码必须可测试、可维护、可 review
- 统一命名、注释、格式、提交规范

## 编码规范
- 变量/函数：见名知意，小驼峰（Python）/ 大驼峰（C++类）/ 下划线（C）
- 注释：关键逻辑、复杂算法、接口契约必须注释；禁止冗余注释
- 格式：自动格式化工具（clang-format、black、yapf），统一行宽80-120

## 工程流程
- 开发前：需求拆解、接口定义、测试用例设计
- 开发中：小步提交、单元测试、静态检查、代码审核
- 发布前：集成测试、性能测试、安全扫描、文档更新

## 质量标准
- 单元测试覆盖率 ≥ 70%（核心模块 ≥ 90%）
- 无高危/中危安全漏洞
- 构建/部署脚本自动化
- 日志/监控/告警完备
